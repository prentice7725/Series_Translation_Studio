# Series Translation Studio — UI/UX 기획서 (화면 설계서)

## 0. 문서 개요

### 0.1 목적

이 문서는 `series_translation_studio_prd_design.md`의 기획·시스템 설계를 바탕으로, 실제 사용자가 보고 조작하는 화면·흐름·상호작용을 정의한다. PRD가 "무엇을 만들 것인가"를 다룬다면 이 문서는 "사용자가 어떻게 쓸 것인가"를 다룬다.

### 0.2 범위

- 데스크톱 앱 (Electron + React) UI 전반
- 주요 화면 레이아웃과 상태
- 사용자 흐름 (Phase 1 / Phase 2 / Phase 3)
- 공통 컴포넌트, 단축키, 외부 전송 동의 UX
- 디자인 토큰 (색/타이포/스페이싱)

범위 밖: 픽셀 단위 디자인 시안, 일러스트레이션, 모바일/웹, 마케팅 자료.

### 0.3 용어

| 용어 | 정의 |
|---|---|
| 프로젝트 | 한 시리즈 단위 작업 공간 (예: 보르코시건) |
| 책 (Book) | 시리즈의 한 권 |
| Phase | PRD의 Phase 0~3 |
| Job | 번역 작업 단위 |
| Segment | 번역 최소 단위 (문단 또는 그보다 작은 블록) |
| Alignment | 영어 원문과 한국어 참고문의 정렬 |
| Chapter Alignment | 챕터 단위 매핑 (사용자 확정 필수) |
| TM | Translation Memory |
| AI 편집장 | Phase 2의 감수 주체 (LLM) |
| spoiler-safe mode | 본문을 미리 보지 않고 EPUB 생성까지 진행하는 모드 |

### 0.4 핵심 사용자 가정

- 1인 사용자, 로컬 데스크톱 사용
- 한국어 UI 기본, 원문은 영어
- 데스크톱 화면 최소 1440 × 900, 권장 1920 × 1080 이상
- 키보드 사용에 익숙함 (단축키 우대)
- 정보 밀도가 높은 전문가용 도구 톤 (CAT 툴, IDE 류)

### 0.5 개정 반영 요약

이번 개정에서는 구현 착수 위험을 낮추기 위해 다음을 반영했다.

```text
- MVP 화면 범위를 MVP-0 / MVP-1 / MVP-2 / MVP-3으로 재분할
- Welcome / Import / Export / 외부 전송 동의에 개인용·DRM·배포 금지 안내 추가
- Review Studio 초기형을 2-pane 최소 감수 화면으로 명확히 분리
- Cost & Usage, Character, Stylebook, AI Editorial 고도화는 MVP 이후 항목으로 이동
- 고정 모델명 예시를 Project default model 중심으로 변경
- LLM Judge 비용은 범위와 보수적 최대치로 표시
```

## 2. 디자인 원칙

1. **밀도 우선, 화려함 후순위.** CAT 툴/IDE 톤. 화면 간 점프를 줄인다.
2. **3-pane은 깨지 않는다.** 원문/번역/참고는 옆으로 나란히. 위아래로 쌓지 않는다.
3. **자동의 결과는 항상 사람이 확인할 초안이다.** 자동 매칭, AI 편집장 감수, QA 모두 승인/거부 가능한 형태로.
4. **확정 단계는 절대 우회하지 않는다.** Chapter Alignment 확정은 명시적 버튼. "자동으로 다음 단계로" 없음.
5. **비용은 상시 표시.** 외부 API 호출의 누적 비용과 추정 잔여 비용을 헤더에 표시.
6. **외부 전송은 사전 동의.** 본문이 외부로 나가는 모든 호출은 범위를 명시하고 동의 받음.
7. **키보드 우선.** 매 segment마다 마우스가 필요하면 감수가 멈춘다. 핵심 동작은 모두 단축키.
8. **중단/재개는 1급 시민.** 모든 긴 작업은 안전하게 중단/재개. 진행률·중단·재개 버튼이 같은 위치.

톤 & 보이스:

- 카피는 사무적이고 짧다. 마케팅 톤·이모지 회피
- 에러 메시지는 "무엇이 일어났는지 + 사용자가 할 수 있는 일" 구조
- 상태값은 한국어로 번역하되 코드 영역에서는 영문 원본 유지

## 3. 공통 컴포넌트 & 패턴

### 3.1 헤더

모든 메인 화면 상단에 고정.

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ ≡  Books › Shards of Honor › Alignment    [🔍]   $ 0.42 / 추정 ~$1.10  🔔│
└──────────────────────────────────────────────────────────────────────────┘
   ↑   ↑                                  ↑    ↑                       ↑
 toggle  breadcrumb                     검색  비용 표시              알림
```

- 사이드바 토글, breadcrumb (클릭 가능), 글로벌 검색(Ctrl+K), 비용 표시, 알림 벨

### 3.2 사이드 패널

Review Studio, Alignment Review에서 우측에 보조 패널. 폭 320~400px, 드래그 조절·collapse 가능.

### 3.3 상태 배지

| 상태값 | 표기 | 색 토큰 |
|---|---|---|
| pending / proposed | 대기 | `--color-state-pending` (회색) |
| running / translating | 진행중 | `--color-state-running` (파랑) |
| needs_review | 검토 필요 | `--color-state-review` (노랑) |
| approved | 승인됨 | `--color-state-approved` (녹색) |
| rejected | 거부됨 | `--color-state-rejected` (적색, 약함) |
| failed / error | 실패 | `--color-state-error` (적색) |
| paused | 일시정지 | `--color-state-paused` (회색, 줄무늬) |

### 3.4 confidence 표시

세 형태로 표시한다.

```text
1) 숫자 + 막대        0.91 ████████░░
2) 색 칩              [높음 0.91] / [중간 0.74] / [낮음 0.52]
3) 막대만 (목록 빽빽)   ████████░░
```

기준: 0.85+ 높음 · 0.65~0.85 중간 · 0.65- 낮음.

### 3.5 매핑 타입 뱃지

`[1:1] [1:N] [N:1] [N:M] [1:0] [0:1]` — 1:0 / 0:1은 옅은 회색 (의역에 의한 누락/추가).

### 3.6 진행률 바

```text
챕터 12 / 24                              45%
████████████░░░░░░░░░░░░░░░░    예상 잔여: 23분 · 잔여 비용: ~$0.38
```

요소: 현재/전체, 백분율, 진행바, 잔여 시간, 잔여 비용.

### 3.7 토스트 / 알림

- **토스트**: 우측 하단, 3초 후 사라짐. 비차단. "Segment 저장됨" 등
- **다이얼로그**: 차단형. 외부 전송 동의 등
- **드로어**: 우측 슬라이드. 비차단. QA Issue 상세 등

### 3.8 외부 전송 동의 다이얼로그

본문이 외부로 나가는 모든 작업의 첫 호출 직전.

```text
┌──────────────────────────────────────────────────────────────┐
│  ⚠  외부 전송 확인                                              │
├──────────────────────────────────────────────────────────────┤
│  이 작업은 다음을 외부 서비스로 전송합니다.                        │
│                                                              │
│  • 대상: Vertex AI (Project default model)                           │
│  • 전송 내용: Shards of Honor 영어 본문 약 5,000 문단           │
│  • 추정 비용: $0.20 ~ $0.60 (보수적 최대 $1.00)                                            │
│  • 예상 소요: 작업량과 provider 상태에 따라 변동                                              │
│                                                              │
│  ☐ 이 책의 같은 종류 호출에서 다시 묻지 않음                       │
│  ☐ 이 프로젝트 전체에서 다시 묻지 않음                           │
│                                                              │
│                       [취소]      [동의하고 시작]                │
└──────────────────────────────────────────────────────────────┘
```

"다시 묻지 않음"은 책별/프로젝트별 두 단계까지. 전체 앱 단계는 안전장치로 제공하지 않는다.

## 4. 사용자 여정

### 4.1 여정 1: Phase 1 — 기존 번역권으로 TM 구축

대표 사용자: "보르코시건 시리즈를 새로 번역하기 전에, 1~2권 정발본을 가지고 시리즈 용어집과 TM을 만들고 싶다."

```mermaid
flowchart TD
    A[프로젝트 생성] --> B[Shards of Honor 영어 EPUB 추가]
    B --> C[Shards of Honor 한국어 EPUB 추가]
    C --> D[Alignment 시작]
    D --> E[자동 전처리]
    E --> F[Body Start 확인]
    F --> H[임베딩 생성]
    H --> I[Chapter LLM Judge]
    I --> J[Chapter Alignment Review]
    J -->|확정| L[Paragraph Alignment]
    L --> M[Paragraph Review]
    M --> N[Low-conf window 재검증]
    N --> O[승인 pair → TM 등록]
    O --> P[Glossary 후보 추출]
    P --> Q[Glossary 확정]
    Q --> R[Stylebook 초안]
```

핵심 길목:

- **Body Start 확인**: 자동 탐지가 거의 맞지만 사용자가 한 번은 봐야 한다
- **Chapter Alignment 확정**: 가장 중요한 결정 지점. 잘못 확정하면 이후 모든 정렬이 어긋남
- **낮은 confidence window 재검증**: 비용 발생. 호출 여부 사용자 선택
- **Glossary 확정**: 시리즈 일관성의 출발점

### 4.2 여정 2: Phase 2 — 기존 한국어판 있는 권의 재번역 (spoiler-safe)

대표 사용자: "정발된 권을 더 좋은 번역으로 다시 만들고 싶다. 본문 내용은 미리 알고 싶지 않다."

```mermaid
flowchart TD
    A[Phase 2 책 추가] --> B[Phase 1 자산 로드]
    B --> C[영어 ↔ 한국어판 Alignment]
    C --> D[reference TM 등록]
    D --> E[AI 초벌 번역]
    E --> F[AI 편집장 감수]
    F --> G[감수 요약 보기 spoiler-safe]
    G --> H[Spoiler-safe EPUB 생성]
    H --> I[외부 앱에서 독서]
    I --> J[사후 보정 모드]
    J --> K[수정 문장만 gold TM 확정]
```

핵심 길목:

- **AI 편집장 감수 실행 전 비용 동의**: 권당 수천 segment × LLM 호출
- **spoiler-safe 진행 화면**: 본문 노출 없이 진행 표시
- **사후 보정 모드**: 완독 후 진입. 책 전체 segment 검색·수정

### 4.3 여정 3: Phase 3 — 미발간권 본번역

대표 사용자: "Cryoburn은 한국에 출간되지 않았다. 시리즈 자산으로 일관되게 번역하고 싶다."

```mermaid
flowchart TD
    A[Phase 3 책 추가] --> B[Phase 1+2 자산 로드]
    B --> C[Translation Setup]
    C --> D[Chapter 단위 자동 번역]
    D --> E[QA 자동 검사]
    E --> F[Review Studio 사람 감수]
    F --> G[최종 EPUB Export]
```


상태: API key 검증 중/실패 표시. 디스크 권한 없으면 다른 위치 권고.

## 13. 추후 검토 사항

구현 단계에서 다시 다룬다.

- **다국어 UI 영문 지원**: 현재는 한국어 UI만. 추후 영문 UI 옵션
- **Plugin / Extension 모델**: 다른 provider, 다른 워크플로우 확장
- **Reader 모드**: Spoiler-safe EPUB을 외부 앱 없이 앱 내에서 읽기
- **Diff 모드**: 같은 segment의 두 버전을 색 강조 비교
- **공동 작업**: 단일 사용자 전제. 다중 작업자 모드는 비목표 유지
- **자동 백업**: 프로젝트 단위 정기 백업 (cloud 미사용 전제 로컬 외부 드라이브 옵션)
